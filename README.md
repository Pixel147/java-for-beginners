## Змінні та константи
### Теорія https://metanit.com/java/tutorial/2.11.php
### Короткий опис:
<p>В Java, змінні це місце, де можна зберігати дані. Вони мають ім'я і тип, так само як і константи. Різниця між ними в тому, що значення змінної може бути змінене в процесі виконання програми, в той час як константа має фіксоване значення і не може бути змінена під час виконання програми. Змінні і константи створюються для того, щоб зберігати і передавати дані між різними частинами програми. Використання констант може допомогти захистити дані від несанкціонованого зміни і зберегти цілісність даних.</p>

## Консольне введення/виведення в Java
### Теорія https://metanit.com/java/tutorial/2.9.php
### Короткий опис
<p>Java має декілька класів та методів для консольного введення та виведення даних. Найпоширеніші з них є:

System.in - стандартний потік введення, який може бути використаний для отримання даних з консолі.
System.out - стандартний потік виведення, який може бути використаний для виведення даних на консоль.
Наприклад
```
System.out.print("Enter your name: ");

```
В цьому прикладі коду спочатку виводиться повідомлення "Enter your name: ", затем користувач вводить своє ім'я і зберігається в змінну name, потім виводиться "Hello, (введене ім'я)!"
Крім того, Java також має клас Scanner, який може бути використаний для обробки тексту з консолі. Можна використовувати його методи для отримання різних типів даних, таких як nextInt(), nextDouble(), nextLine().</p>

## Арифметичні операції
### Теорія https://metanit.com/java/tutorial/2.3.php
### Короткий опис:
<p>
Java підтримує стандартні арифметичні операції, такі як:

Додавання: +
Віднімання: -
Множення: *
Ділення: /
Залишок від ділення: %
Наприклад:

```
int x = 5;
int y = 3;
int sum = x + y; // sum = 8
int diff = x - y; // diff = 2
int product = x * y; // product = 15
int quotient = x / y; // quotient = 1
```

```
x += y; // equivalent to x = x + y;
x *= y; // equivalent to x = x * y;
```
Java також має клас Math з багатьма методами для виконання більш складних математичних операцій, таких як округлення, обчислення тригонометричних функцій і т.д.

## Умовні вирази
### Теорія https://metanit.com/java/tutorial/2.14.php
### Короткий опис:
<p>
Операції порівняння в Java дозволяють порівнювати два значення і повертають булеве значення (true або false). 
Вони можуть бути використані для створення умовного оператора або циклу. 
Наприклад:

```
int a = 5;
int b = 7;

System.out.println(a == b);   // false
System.out.println(a != b);   // true
System.out.println(a > b);    // false
System.out.println(a < b);    // true
System.out.println(a >= b);   // false
System.out.println(a <= b);   // true
```
Логічні операції в Java дозволяють комбінувати булеві вирази і повертають також булеве значення. 
Вони можуть бути використані для створення складного умовного оператора. Наприклад:
```
boolean x = true;
boolean y = false;

System.out.println(x && y);  // false
System.out.println(x || y);  // true
System.out.println(!x);     // false
```
`&&` - оператор "та", повертає true якщо обидва вирази є true.<br/>
`||` - оператор "або", повертає true якщо хоча б один з виразів є true.<br/>
`!` - оператор "не", змінює булеве значення на протилежне.<br/>
Ці оператори можуть бути використані разом із умовними операторами такими як if та while, щоб створювати складні умови.
</p>

## Конструкція if/else/else if
### Теорія https://metanit.com/java/tutorial/2.3.php
### Короткий опис:
<p>
`if` - це умовний оператор, який дозволяє виконувати блок коду лише якщо вираз в його умові є true. 
<br/>Наприклад:

```
int number = 5;
if (number > 0) {
   System.out.println("number є додатнім");
}
//output : number є додатнім
```

`else` - це оператор, який дозволяє виконувати інший блок коду, якщо умова в if операторі є false. <br/>Наприклад:

```
int number = -5;
if (number > 0) {
   System.out.println("number є додатнім");
} else {
   System.out.println("number є від'ємним або нулем");
}
//output : number є від'ємним або нулем
```

`else if` - це додатковий оператор, який може бути використаний з if для перевірки додаткових умов. Наприклад:

```
int number = 0;
if (number > 0) {
   System.out.println("number є додатнім");
} else if (number < 0) {
   System.out.println("number є від'ємним");
} else {
   System.out.println("number є нулем");
}
```

Як видно з прикладу, умова `if` перевіряє чи number більше нуля, якщо ця умова не виконується, то переходиться до `else if`, яка перевіряє чи number менше нуля. Якщо ні одна із цих умов не виконується, то виконується код у `else`.
Важливо зауважити, що програма буде виконувати тільки перший блок коду який співпадає з умовою, тому `рекомендується об''являти умови від найбільш специфічної до найбільш загальної.
</p>

## Цикли(for,while,do while)
### Теорія https://metanit.com/java/tutorial/2.6.php
### Короткий опис:
<p>
Цикли у Java - це конструкції, які дозволяють виконувати однаковий код декілька разів.

for цикл має структуру

```
for (налаштування початкових значень; умова виконання циклу; крок) {
   тіло циклу;
}
```

while цикл має структуру

```
while (умова) {
   тіло циклу;
}
```

do-while цикл має структуру

```
do {
   тіло циклу;
} while (умова);
```

Цикл <code>for</code> і <code>while</code> виконують тіло циклу, якщо умова виконання циклу є true, а цикл do-while виконує тіло циклу принаймні один раз перед перевіркою умови виконання.
</p>

## Масиви
### Теорія https://metanit.com/java/tutorial/2.4.php
### Короткий опис:
<p>
Масиви у Java - це контейнери, які можуть містити однотипові елементи. Вони можуть бути створені для зберігання будь-якого типу даних (цілі числа, символи, об'єкти і т.д.). Масив має фіксовану довжину, яка встановлюється при створенні масиву, і не може бути змінена пізніше.
<br/>
Масив може бути створений за допомогою ключового слова new та типу даних, а також повного розміру чи заповнення за замовчуванням. Потім можна доступити до елементів масиву за допомогою індексу, який починається з 0.
   
```
int[] myArray = new int[5]; // створення масиву розміром 5 за замовчуванням значення 0

myArray[0] = 10; // заповнення першого елементу масиву значенням 10

int secondElement = myArray[1]; // отримання значення другого елементу масиву
```

<p>
До масивів також можна застосовувати команди такі як цикли для перебору елементів масиву, методи типу length щоб отримати довжину масиву, методи типу sort щоб відсортувати елементи масиву і т.д.
</p>

## Методи
### Теорія <br/> https://metanit.com/java/tutorial/2.7.php <br/> https://metanit.com/java/tutorial/2.16.php <br/> https://metanit.com/java/tutorial/2.17.php
### Короткий опис
<p>
Функція - це блок коду, який виконує певну послідовність операцій та може повертати результат. Функції можуть бути використані для обробки даних, здійснення операцій над об'єктами та організації коду в більш структурованій формі. В Java функції визначаються як методи, в яких можна вказувати аргументи та тип значення яке повертає метод
</p>

```
public class Main {
    public static void main(String[] args) {
        int result = add(5, 10);
        System.out.println("The result is: " + result);
    }

    private static int add(int a, int b) {
        return a + b;
    }
}
```

<p>
Цей код містить функцію add, яка приймає два цілих числа та повертає їх суму. В тілі main метода, ми викликаємо функцію add, передаючи їй значення 5 та 10 як аргументи. Результат виклику функції записується у змінну result і виводиться на консоль.
</p>

## Обробка виключень
### Теорія https://metanit.com/java/tutorial/2.10.php

<p>
Try-catch - це механізм обробки помилок в Java. Використовується для відловлення та обробки виключень (exceptions) в коді. Блок "try" містить код, який може викликати виключення, а блок "catch" містить код, який виконується, якщо виключення виникає. Також може бути доданий блок "finally", який виконується незалежно від того, чи виникло виключення чи ні.
</p>

## Модифікатори доступу та інкапсуляція
### Теорія https://metanit.com/java/tutorial/3.3.php
### Короткий опис

<p>

`public` – клас, метод, змінна з модифікатором public доступні з будь-якого місця програми.

`private` - клас, метод, змінна з модифікатором private, доступні тільки всередині самого класу.

`protected` - клас, метод, змінна з модифікатором protected доступні всередині класу та його спадкоємців.

Модифікатори доступу допомагають обмежити доступ до даних та методів усередині класу та захистити їх від неправильного використання.

</p>
<p>

Інкапсуляція в Java - це механізм, який захищає дані і методи класу від неавторизованого доступу і зміни. <br/>
Вона досягається шляхом використання модифікаторів доступу, таких як private, protected і public щоб обмежити доступ до членів класу. <br/>
Наприклад, змінна, оголошена з модифікатором private буде доступна тільки всередині самого класу і не буде доступна для інших класів або об'єктів.<br/>
Інкапсуляція дозволяє контролювати доступ до даних і методів, таким чином забезпечує безпеку і стабільність роботи коду.<br/>

</p>

## Статичні члени та модифікатор static
### Теорія https://metanit.com/java/tutorial/3.4.php
### Короткий опис

<p>
Статичні члени, методи<br/>

Статичні члени в Java - це класи, методи і змінні, які належать класу, а не конкретному об'єкту. Вони можуть бути викликані без створення екземпляру класу.
Статичні змінні зазвичай використовуються для хранення загальної інформації, яка є однаковою для всіх екземплярів класу. Статичні методи можуть бути викликані без створення екземпляру класу, тому їх часто використовують для виконання дій, які не потребують доступу до даних екземпляру.
<br/>
Ключове слово static використовується для оголошення статичного методу чи змінної.Наприклад:

```
class MyClass {
    static int x = 10;
    static void myMethod() {
        // code here
    }
}
```

Статичні елементи можуть бути викликані за допомогою імені класу і точкою, наприклад:

```

MyClass.x; // 10
MyClass.myMethod();

```

<br/>Статичні константи<br/>
Статичні константи в Java - це змінні, які мають статичний тип і постійне значення. Вони зазвичай оголошуються з модифікатором public static final. Вони можуть бути ініціалізовані тільки один раз, при оголошенні або в конструкторі статичного блоку.<br/>

Наприклад:

```

public class MyClass {
    public static final int MAX_VALUE = 100;
    public static final String GREETING = "Hello";
}

```

Використовуючи статичні константи, ви можете забезпечити, що ваш код буде мати стабільні значення, які не можуть бути змінені під час виконання програми. Вони також дозволяють легко змінювати значення в одному місці коду, щоб відразу ж оновити їх у всіх місцях, де вони використовуються. Також, вони можуть бути використані для забезпечення читабельності коду і зрозумілісті для інших розробників, що будуть працювати з вашим кодом.

<br/>Статичні ініціалізатори<br/>

Статичні ініціалізатори в Java - це блоки коду, які виконуються один раз при завантаженні класу. Вони можуть бути використані для ініціалізації статичних змінних і виконання дій, які необхідні для підготовки класу до роботи. Вони зазвичай оголошуються з модифікатором static.

```
public class MyClass {
    static int x;
    static {
        x = 10;
    }
}
```
Статичні ініціалізатори можуть бути використані для ініціалізації статичних змінних з даними, які можуть бути отримані тільки під час виконання програми, або для завантаження даних з файлу або бази даних. Вони також можуть бути використані для виконання будь-якої початкової конфігурації або налаштування класу, яке потрібно для роботи.
</p>

## Об'єкти яе параметри функцій
### Теорія https://metanit.com/java/tutorial/3.14.php
### Короткий опис

<p>
В Java, об'єкт може бути параметром функції. Це означає, що функція може приймати об'єкт як вхідний параметр і використовувати його для виконання своїх дій.<br/>
Приклад:

```
class MyClass {
  int x;
  int y;
  MyClass(int x, int y) {
    this.x = x;
    this.y = y;
  }
}

public class Main {
  public static void main(String[] args) {
    MyClass obj = new MyClass(10, 20);
    myFunction(obj);
    //output: 10 20
  }

  public static void myFunction(MyClass o) {
    System.out.println(o.x + " " + o.y);
  }
}
```

В цьому прикладі, об'єкт obj класу MyClass є параметром функції myFunction і виводить значення полів x і y.
</p>

## Внутрішні та вкладені класи
### Теорія https://metanit.com/java/tutorial/3.12.php
### Короткий опис

<p>
В Java, існують два типи вкладених класів: внутрішні та вкладені.

Внутрішні класи (Inner Classes) є класами, які оголошені всередині іншого класу. Вони мають доступ до всіх полів та методів свого зовнішнього класу та можуть бути оголошені як public, private чи protected.

Вкладені класи (Nested Classes) є класами, які оголошені в межах іншого класу, але не мають доступу до полів свого зовнішнього класу. Вони можуть бути оголошені як static або non-static.

```

class OuterClass {
   private int x = 10;

   class InnerClass {
      public void accessOuterClass() {
         System.out.println("x = " + x);
      }
   }
}

class NestedClass {
   static class StaticNestedClass {
      public void myMethod() {
         System.out.println("Static nested class method");
      }
   }
}

```

В цьому прикладі, клас InnerClass є внутрішнім класом та має доступ до приватного поля x класу OuterClass, а StaticNestedClass є вкладеним класом та не має доступу до полів класу NestedClass.
</p>

```
abstract class Shape {

}
```

## Інтерфейси
### Теорія https://metanit.com/java/tutorial/3.7.php
### Короткий опис

<p>
Інтерфейси у Java - це спеціальний тип класів, які задають лише сигнатури методів (не мають реалізації) і статичні константи. Класи, які імплементують інтерфейс, обов'язані реалізувати всі його методи. Інтерфейси можуть бути використані для опису спільного інтерфейсу для групи класів, які можуть бути інтерчейнд (для різних класів) і можуть бути використані для поліморфізму, дозволяючи викликати методи, які мають однакову сигнатуру для різних типів об'єктів. Інтерфейси можуть бути реалізовані багатьма класами, а класи можуть реалізувати багато інтерфейсів.
</p>

### Різниця між ітерфейсами та абстрактними класами

<p>
Різниця між інтерфейсами та абстрактними класами в Java полягає в тому, що:<br/>
Інтерфейси містять лише сигнатури методів (не мають реалізації) і статичні константи. Класи, які імплементують інтерфейс, обов'язані реалізувати всі його методи.<br/>
Абстрактні класи можуть містити реалізації методів, а також можуть містити поля та конструктори. Класи, які успадковують абстрактний клас, можуть використовувати ці реалізації або перевизначити їх.<br/>
Інтерфейси можуть бути імплементовані кількома класами, а клас може реалізувати багато інтерфейсів. Абстрактні класи ж можуть бути успадковані лише одним класом.
</p>
